## Rule
The first `<p>` element inside the main content area should be at least 20 characters of substantive text. Shorter first paragraphs get an INFO signal — not a failure, just a hint that the page isn't "answer-first."

## Why (rationale)
ICP-2 v2.1 values answer-first body structure: a reader arriving from search or an AI summary should immediately see a direct answer to the implied question, not navigation chrome, a "Welcome to..." throat-clearing, or an image caption. 20 chars is a very lenient floor; real answer-first paragraphs are usually 80-200.

Rule is severity `info` because catching all edge cases (image captions that legitimately come first, non-article page types) would produce too many false positives at higher severity.

## Pass example
```html
<main>
  <article>
    <p>Nassau cruise day decisions come down to three choices: walk downtown to Bay Street, take a jitney to Cable Beach, or taxi to Atlantis. Each pairs with a different pace of day and budget.</p>
    ...
```

## Fail example
```html
<main>
  <article>
    <p>Hello.</p>
    <p>Welcome to our Nassau cruise port page!</p>
    ...
```
First paragraph is 6 chars ("Hello."). Validator emits: `First paragraph is very short — consider leading with a direct answer about this port`.

## Fix guidance
Lead with the page's answer. If the answer-line element (see SHIP-001's ship-page version) already provides this, make sure it IS the first `<p>` inside `main`. Don't bury the answer under "Welcome" or rhetorical questions ("Planning a Nassau cruise?").
