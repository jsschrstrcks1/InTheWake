## Rule
Ship page carousels (typically Swiper.js instances with class `.swiper` or `.carousel`) must declare an `aria-label` identifying what the carousel contains ("Photo gallery", "Ship deck plans", "Guest reviews").

## Why (rationale)
Screen readers announce the carousel landmark but don't describe what's inside. Without `aria-label`, users tabbing into the region hear "region" and then a series of slides with no context. A label turns the region into "Photo gallery, region, slide 1 of 10..." — immediately usable.

Ship validator enforces; port and venue validators don't. Standards silent — backfill needed.

## Pass example
```html
<div class="swiper ship-photo-carousel" aria-label="Allure of the Seas photo gallery">
  <div class="swiper-slide">...</div>
  ...
</div>
```

## Fail (warning) example
```html
<div class="swiper ship-photo-carousel">
  <div class="swiper-slide">...</div>
</div>
```
Validator emits: `Carousel N missing aria-label`.

## Fix guidance
Add a descriptive `aria-label` naming the carousel's content. One phrase. Don't repeat it on every slide — the label applies to the whole region.
