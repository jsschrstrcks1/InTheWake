## Rule
A port hero section must contain at least one `<img>` element. Text-only "heroes" (CSS background-image, color blocks) are not heroes — a hero is a photograph.

## Why (rationale)
The hero carries most of the page's emotional signal in the first fold. CSS background images are invisible to screen readers by default and can't carry alt text; they're inferior for accessibility (A11Y) and for content verification (IMG Verification Protocol). An `<img>` element is both a11y-friendly and inspectable.

## Pass example
```html
<section class="port-hero">
  <img src="/ports/img/nassau/nassau-hero.webp" alt="..." width="1800" height="900"/>
  <div class="port-hero-overlay"><h1>Nassau, Bahamas</h1></div>
</section>
```

## Fail example
```html
<section class="port-hero" style="background-image: url(/ports/img/nassau/nassau-hero.webp);">
  <h1>Nassau, Bahamas</h1>
</section>
```
Validator emits: `Hero box must contain an image`. No `<img>` descendant found.

## Fix guidance
Use an `<img>` element. If you need the image to fill the section edge-to-edge, set `.port-hero img { width: 100%; height: 100%; object-fit: cover; }` rather than falling back to `background-image`.
