## Rule
For every port HTML file `ports/<slug>.html`, a directory `ports/img/<slug>/` must exist on disk. Absence fails as BLOCKING. This is the filesystem-level structural rule — the validator can't verify image content if the directory doesn't exist.

## Why (rationale)
Per-port image directories are the site convention for clean separation. Shared images across ports (via symlinks or direct duplication) are handled separately via cross-port-duplication rules (IMG-015) and the `allowedDuplicates` file. Without the per-port directory, images live in an ad-hoc location and cross-port audits can't run.

## Pass example
```
ports/nassau.html          ← port page
ports/img/nassau/          ← directory exists
  nassau-hero.webp
  nassau-hero-attr.json
  nassau-cable-beach.webp
  nassau-cable-beach-attr.json
  ...
```

## Fail example
```
ports/nassau.html          ← port page exists
<no ports/img/nassau/ directory>
```
Validator emits: `Port image directory not found: ports/img/nassau/`. Validation halts — downstream port-image rules can't run.

## Fix guidance
Create the directory before the port page goes live. If images were put elsewhere (e.g., `assets/ports/nassau/`), move them to the convention: `ports/img/<slug>/`.
