## Rule
The visible footer Soli Deo Gloria dedication element must NOT carry `aria-hidden="true"`. Screen readers must be able to read the invocation. The top-of-file SDG HTML comment (THEO-001) is separate — that's a source comment, not visible DOM. This rule is about the visible footer dedication.

## Why (rationale)
**The 2026-02 audit escape.** 226 ship pages were found with `aria-hidden="true"` on the visible footer SDG element — hiding the theological invocation from screen readers. Per pastoral guardrails and the project's theological foundation, the invocation must be accessible to all users including those using assistive technology.

**Related but distinct issue:** the class-naming mistake caught in `CAREFUL.md` Session Learnings — `.visually-hidden` was wrongly applied to a dedication element that needed `.dedication-hidden` (truly hidden, aria-hidden="true") instead. That's a different case — the decorative dedication above the footer, which IS meant to be hidden from screen readers. The visible footer dedication caught by this rule (A11Y-011) must remain accessible.

## Pass example
```html
<footer>
  <p class="sdg-dedication">Soli Deo Gloria — To God alone be the glory.</p>
</footer>
```

## Fail example
```html
<footer>
  <p class="sdg-dedication" aria-hidden="true">Soli Deo Gloria — To God alone be the glory.</p>
</footer>
```
Validator emits: `Soli Deo Gloria footer dedication has aria-hidden="true" — should be accessible to all users`.

## Fix guidance
Remove `aria-hidden="true"` from the footer SDG element. If the element is intentionally decorative-only (not the visible dedication), distinguish it with a different class (`.dedication-hidden` for the truly-hidden variant per CAREFUL.md) — but the visible dedication must stay accessible.

## Severity note
Validator currently emits this at `WARNING` (ship-page-v2.js line 2556). Given the pastoral importance and the documented 226-page escape, an argument could be made for promoting to BLOCKING in the future — but that's a judgment call and is out of scope for this rule's extraction. Noted here for future consideration.
