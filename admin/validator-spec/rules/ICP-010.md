## Rule
Pages must NOT include `<meta name="geo.region">` or `<meta name="geo.placename">` tags. These are legacy SEO artifacts that no modern search engine uses. Geographic context belongs in JSON-LD `Place` entities, not ancient meta tags.

## Why (rationale)
Another ICP-2 v2.1 anti-pattern. `geo.region` and `geo.placename` were part of a brief mid-2000s attempt at geo-targeting metadata; they were never widely adopted by crawlers and are now pure cruft. Including them signals a stale SEO template was copied. Real geography belongs in `mainEntity` JSON-LD (see SCHEMA-001).

## Pass example
```html
<!-- no geo.* meta; geography lives in JSON-LD Place entity -->
<script type="application/ld+json">
{ "@type": "Place", "geo": { "@type": "GeoCoordinates", "latitude": 25.0838, "longitude": -77.3412 } }
</script>
```

## Fail example
```html
<meta name="geo.region" content="BS-NP"/>
<meta name="geo.placename" content="Nassau"/>
```
Validator emits: `geo.region/geo.placename meta tags are irrelevant — remove them`.

## Fix guidance
Delete both tags. If you need geographic context, add it to the `mainEntity` JSON-LD `Place` object (ICP-2 and Schema.org both recognize this path).
