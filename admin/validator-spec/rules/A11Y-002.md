## Rule
Every HTML page must declare `<meta charset="utf-8">` (or any UTF-8 equivalent) in `<head>`. The validator does a literal string match for `<meta charset="` — any form satisfies as long as the tag is present.

## Why (rationale)
Without an explicit charset declaration, browsers fall back to locale-specific defaults. That's fine in the Americas; it breaks diacritics and CJK characters silently for other visitors. A11Y overlaps with internationalization — both require the tag.

## Pass example
```html
<head>
  <meta charset="utf-8"/>
  ...
</head>
```

## Fail example
```html
<head>
  <!-- no meta charset -->
  <title>...</title>
</head>
```
Validator emits: `Missing <meta charset=""> tag`.

## Fix guidance
Add `<meta charset="utf-8"/>` as the first child of `<head>`. It must appear before any content-dependent tags (like `<title>`) so the browser has the encoding before parsing them.
