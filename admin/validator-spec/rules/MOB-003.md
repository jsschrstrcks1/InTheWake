## Rule
A hero image must be contained by its parent on narrow viewports. The validator flags hero `<img>` elements that declare fixed pixel dimensions (e.g., `width="1800"`) without a CSS strategy to prevent viewport overflow (`object-fit: cover`, `max-width: 100%`, parent `overflow: hidden`, etc.).

## Why (rationale)
A 1800×900 hero image on a 375px phone viewport overflows by 1425px. Most browsers handle this by sliding horizontally — making the rest of the page scroll sideways too. The fix is simple CSS; the problem is just remembering to apply it on every hero.

## Pass example
```html
<section class="port-hero">
  <img src="..." alt="..." width="1800" height="900"/>
</section>
<style>
  .port-hero img { width: 100%; height: auto; object-fit: cover; }
</style>
```

## Fail (warning) example
```html
<section class="port-hero">
  <img src="..." alt="..." width="1800" height="900"/>
</section>
<!-- no CSS constraining the image on mobile -->
```
Validator emits: `1 hero image(s) with containment issues: <img> width=1800 without responsive CSS`.

## Fix guidance
Ensure `.port-hero img`, `.hero img`, or `.ship-hero img` has `max-width: 100%` in the shared stylesheet. This is typically already set globally for `img` elements — the rule catches hero-specific overrides that break the default.
