## Rule
Every `<figure>` on a port page must contain a `<figcaption>` element, and that figcaption must contain at least one `<a>` link — typically the source-URL credit link (Wikimedia, NPS, photographer's page).

## Why (rationale)
Inline visible attribution is how the site meets CC-BY-SA and CC-BY license obligations. The attr.json companion file (ATTR-001) tracks provenance for the repo; the figcaption anchor makes the credit visible to readers. Both are required.

## Pass example
```html
<figure>
  <img src="/ports/img/nassau/prince-george-wharf.webp" alt="Cruise ship at Prince George Wharf"/>
  <figcaption>Prince George Wharf, Nassau — photo <a href="https://commons.wikimedia.org/wiki/File:PGW.jpg">Wikimedia Commons / CC-BY-SA</a></figcaption>
</figure>
```

## Fail example
```html
<figure>
  <img src="/ports/img/nassau/prince-george-wharf.webp" alt="..."/>
  <figcaption>Prince George Wharf, Nassau</figcaption>
</figure>
```
Figcaption has no anchor. Validator emits: `N images missing photo credits in figcaption`.

## Fix guidance
Add the source link inside figcaption. If the image is your own photography, link to an author/about page or to `/about-us.html`. Don't leave figcaption as bare prose.
