## Rule
Text content elements must not have inline `font-size` values below 15px. The validator scans inline styles and flags body-text elements (paragraphs, list items, links inside prose) that fall below the floor. Chrome elements (breadcrumbs, footer micro-copy, navigation labels) are exempt; the Soli Deo Gloria invocation is explicitly exempt because its small stylized rendering is intentional and covered by Standards Section 2.3.

## Why (rationale)
15px is roughly the smallest size that reads comfortably on a mobile device without squinting or pinch-zoom. The site serves older readers, grieving travelers, and readers in stressful planning moments — making them work to read body text is a real failure. The 15px floor is adopted site-wide as the minimum readability bar.

## Pass example
```html
<p style="font-size: 16px;">Content body text.</p>
<p>Content body text without inline override (inherits site base ≥ 15px).</p>
<nav class="breadcrumb-nav" style="font-size: 13px;">...</nav>  <!-- exempt: breadcrumb nav -->
```

## Fail (warning) example
```html
<p style="font-size: 12px;">Fine print that someone put inline.</p>
<li style="font-size: 11px;">List item</li>
```
Validator emits: `2 element(s) with font-size below 15px floor: <p> font-size: 12px — "Fine print..." | <li> font-size: 11px — "List item"`.

## Fix guidance
Remove the inline font-size. If the intent was visually-smaller text (a caption, a note), use a CSS class (`.note` at 14px with an explanation comment about why it's exempt) rather than inline override. Inline font-size below 15px is almost always a mistake.
