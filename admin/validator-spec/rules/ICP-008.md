## Rule
Every page must include a `<link rel="canonical" href="..."/>` element pointing to the page's canonical URL. See ICP-016 for the requirement that the canonical be an absolute URL on `cruisinginthewake.com`.

## Why (rationale)
Canonical URLs prevent duplicate-content SEO penalties when a page is reachable via multiple paths (trailing slash, query strings, HTTP vs HTTPS). The site is a single-origin static build, but search engines still need the explicit declaration.

## Pass example
```html
<link rel="canonical" href="https://cruisinginthewake.com/ports/nassau.html"/>
```

## Fail example
```html
<!-- canonical missing -->
```
Validator emits: `Missing <link rel="canonical"> — required for SEO`.

## Fix guidance
Add the canonical link in `<head>`. Use the page's full absolute URL on cruisinginthewake.com. See ICP-016 for the absolute-URL requirement.
