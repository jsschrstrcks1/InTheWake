## Rule
`last-reviewed` content must match exactly `YYYY-MM-DD` (ISO 8601 date, no time, no timezone). `2026-04-15` passes; `April 15, 2026`, `15-04-2026`, `2026-4-15`, or `2026-04-15T10:30:00Z` all fail.

## Why (rationale)
Consistency. The validator parses this field for staleness checks (ICP-007) and JSON-LD mirroring (ICP-013). A free-form date means either complex parsing logic or regular breakage. ISO 8601 is unambiguous across every locale and AI consumer.

## Pass example
```html
<meta name="last-reviewed" content="2026-04-15"/>
```

## Fail example
```html
<meta name="last-reviewed" content="April 15, 2026"/>
<meta name="last-reviewed" content="2026/04/15"/>
<meta name="last-reviewed" content="15-04-2026"/>
```
Validator emits: `last-reviewed must be YYYY-MM-DD format, found "..."`.

## Fix guidance
Use ISO 8601 date strings. No time, no timezone. If you need more precision (rare), open a discussion — don't invent a format.
