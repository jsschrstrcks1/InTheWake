## Rule
The hero section must contain the port's name as an overlay — either an `<h1>`, `.port-hero-overlay`, or `.port-name-overlay` element. Hero-as-image-only without a name overlay fails.

## Why (rationale)
An image alone can be ambiguous (many cruise ports have similar waterfronts). The name overlay anchors the hero to the specific port visually and semantically. Also: search engines and AI consumers use the `<h1>` inside the hero as strong signal for page subject.

Standards docs describe "hero with title" at the gestural level but don't name the specific `.port-hero-overlay` / `.port-name-overlay` classes. Backfill required.

## Pass example
```html
<section class="port-hero">
  <img src="..." alt="..."/>
  <div class="port-hero-overlay">
    <h1>Nassau, Bahamas</h1>
    <p class="subtitle">Cable Beach, Atlantis, Junkanoo, and the 5-minute walk to Bay Street</p>
  </div>
</section>
```

## Fail example
```html
<section class="port-hero">
  <img src="..." alt="Nassau cruise port"/>
  <!-- no h1, no overlay div -->
</section>
<h1>Nassau, Bahamas</h1> <!-- h1 present but OUTSIDE hero; doesn't satisfy rule -->
```
Validator emits: `Hero box must contain port name overlay (h1 or .port-hero-overlay)`.

## Fix guidance
Put the `<h1>` inside the hero section, typically wrapped in `.port-hero-overlay` with positioning CSS (`position: absolute; bottom: 0; ...`). If you already have the h1 elsewhere, move it into the hero — don't duplicate it.
