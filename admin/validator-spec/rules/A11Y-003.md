## Rule
Every HTML page must declare `<meta name="viewport" content="width=device-width, initial-scale=1">`. The validator checks for presence of the `name="viewport"` attribute; content form isn't validated character-for-character here, but the canonical pattern is `width=device-width, initial-scale=1`.

## Why (rationale)
Without `viewport`, mobile browsers render at a simulated 980px width, zoom out to fit, and produce tiny unreadable text. Users can pinch-zoom but every interaction becomes labor. Real mobile users abandon such pages instantly.

Tightly related to MOB-005 (touch targets) and mobile-readiness checks. Viewport presence itself is also validator-tracked as MOB-001 in validate-mobile-readiness.js — this A11Y rule and that MOB rule overlap on the same check.

## Pass example
```html
<head>
  <meta charset="utf-8"/>
  <meta name="viewport" content="width=device-width, initial-scale=1"/>
  ...
</head>
```

## Fail example
```html
<head>
  <meta charset="utf-8"/>
  <!-- no viewport — mobile renders at 980px simulated width -->
</head>
```
Validator emits: `Missing <meta name="viewport"> tag`.

## Fix guidance
Add the tag. The canonical content is `width=device-width, initial-scale=1`. Avoid `user-scalable=no` — it disables pinch-zoom, which is an accessibility regression for low-vision readers.
