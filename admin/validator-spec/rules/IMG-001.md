## Rule
Every `<img>` element on the site must have an `alt` attribute with content that describes the image, OR be explicitly marked as decorative via `role="presentation"` or `aria-hidden="true"`. Empty alt on a content image is a failure.

## Why (rationale)
WCAG 2.1 AA Success Criterion 1.1.1. Screen readers depend on alt text. The site serves disabled travelers — a blank alt is both a legal compliance failure and a failure of the site's stated mission. Validator enforces at commit.

## Pass example
```html
<img src="/assets/ports/nassau/prince-george-wharf.webp"
     alt="Royal Caribbean ship docked at Prince George Wharf with downtown Nassau and Senor Frog's visible in background"
     width="1200" height="800" loading="lazy"/>

<!-- Decorative icon, correctly hidden from screen readers: -->
<img src="/assets/icons/chevron-right.svg" alt="" aria-hidden="true"/>
```

## Fail example
```html
<img src="/assets/ports/nassau/hero.webp"/>

<img src="/assets/ports/nassau/beach.webp" alt=""/>
```
Both fail: first has no alt attribute; second has empty alt on a non-decorative content image.

## Fix guidance
Write an alt that describes what the image shows and why a reader would care. Generic alts ("port image", "beach photo") technically pass the attribute check but fail WCAG's quality intent — prefer specific, descriptive text. For genuinely decorative chrome (dividers, icons alongside labeled text), use `alt="" aria-hidden="true"`.
