## Rule
An image declared decorative via `aria-hidden="true"` or `role="presentation"` should have `alt=""` (empty). The validator skips the alt-text length check on these (they're treated as exempt), but the two declarations should agree: either the image is decorative (aria-hidden + empty alt) or informational (no aria-hidden + descriptive alt).

## Why (rationale)
Contradictory declarations confuse assistive tech. `aria-hidden="true" alt="Beautiful sunset"` tells screen readers "skip this image" but also declares an alt text that would be read IF the image weren't hidden. The conflict is benign in practice (aria-hidden wins) but signals someone forgot to finish the decision.

The validator is permissive today (it exempts aria-hidden images from alt checks). This rule captures the consistency expectation — a future validator enhancement could warn on the contradiction.

## Pass example
```html
<!-- Decorative icon -->
<img src="/assets/icons/chevron.svg" alt="" aria-hidden="true"/>

<!-- Informational photo -->
<img src="/ports/img/nassau/hero.webp" alt="Royal Caribbean ship at Prince George Wharf"/>
```

## Fail example (consistency warning)
```html
<img src="/assets/icons/chevron.svg" alt="Right-facing chevron" aria-hidden="true"/>
```
Aria-hidden declares decorative; alt text suggests informational. Pick one.

## Fix guidance
If the image is decorative: `alt="" aria-hidden="true"`. If it's informational: descriptive alt, no aria-hidden. The validator doesn't block on mixed declarations today but the internal consistency matters for readability.
