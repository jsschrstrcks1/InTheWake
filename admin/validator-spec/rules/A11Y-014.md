## Rule
The `<html>` element must declare a `lang` attribute with a non-empty value (e.g., `lang="en"`). Required by WCAG 2.1 AA SC 3.1.1.

**Note on implementation gap:** The port validator (`validate-port-page-v2.js` line 3717-3719) checks charset but does NOT currently check lang. The rule is enforced via `.claude/standards/html.yml` declaration. This is a gap worth closing — future port/ship/venue validator work should add the lang check alongside the existing charset/viewport checks.

## Why (rationale)
Screen readers use `lang` to select the correct pronunciation engine. An unlabeled English page read with a French engine produces nonsense. Automated translators, language-aware spell-check, and AI summary tools all rely on the declaration.

## Pass example
```html
<!doctype html>
<html lang="en">
<head>
  ...
```

## Fail example
```html
<!doctype html>
<html>
<head>
  ...
```
Standards YAML flags as error. Port validator does NOT currently check this (gap — see above).

## Fix guidance
Add `lang="en"` to `<html>`. For pages with significant content in another language (none currently on this site), use the appropriate ISO 639-1 code.
