## Rule
Inline-styled widths greater than 480px are flagged. On a 375px-wide phone viewport, a `width: 600px` block forces horizontal scroll and breaks the mobile reading experience. The rule is at warn severity because some legitimate uses exist (wide tables in a scrollable container — see MOB-004) that the heuristic doesn't distinguish.

## Why (rationale)
A fixed large-px inline width is a reliable tell that the author was thinking about desktop and forgot about mobile. Fluid widths (`%`, `max-width`, `min(100%, 900px)`) adapt; fixed wide widths don't. 480px is a useful threshold because it's the iPhone SE landscape width, common phone portrait widths, and a commonly-used small-screen breakpoint.

## Pass example
```html
<div class="wide-content" style="max-width: 900px; margin: 0 auto;">
  ...
</div>
```
`max-width` instead of `width` — content fills narrower viewports.

## Fail (warning) example
```html
<figure style="width: 720px;">
  <img src="..." alt="..."/>
</figure>
```
Validator emits: `1 element(s) with inline width > 480px: <figure> style="width: 720px;"`.

## Fix guidance
Replace `width: Npx` with `max-width: Npx; width: 100%;`, or move the constraint to a CSS class with a `@media (max-width: Npx)` fluid fallback. Keep the wide layout on desktop, let it collapse on phones.
