## Rule
Every page should include Twitter Card meta tags with `twitter:card` set to `summary_large_image`. This controls appearance when shared on X (Twitter), Bluesky, and other platforms that read Twitter Card markup. No validator enforces — orphan.

## Why (rationale)
`summary_large_image` renders the hero image prominently in the share card. The default `summary` card shows a small thumbnail. For a visual-first travel planning site, the large image card is the correct presentation.

## Fix guidance
Add to `<head>`: `<meta name="twitter:card" content="summary_large_image"/>`, `<meta name="twitter:title" content="..."/>`, `<meta name="twitter:description" content="..."/>`. Can reuse og:image for the image (most platforms fall back to OG if twitter:image is missing).
