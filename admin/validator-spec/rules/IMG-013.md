## Rule
The port's image directory (see IMG-012) must contain at least one image file (`.webp`, `.jpg`, or `.png`). An existing-but-empty directory fails as BLOCKING.

## Why (rationale)
An empty image directory means the port page's image references are either broken (pointing at missing files) or the page has no images at all (failing IMG-005's minimum 11). Either way, the page isn't ready to ship. Catching this at the directory level produces a clearer error than letting IMG-005 count to zero.

## Pass example
`ports/img/nassau/` contains 15 .webp files. Passes.

## Fail example
`ports/img/nassau/` is an empty directory, or contains only README.md. Validator emits: `No images found in ports/img/nassau/`.

## Fix guidance
Source real, attributed, verified images before the port page goes live. See ATTR-001 for attribution, ATTR-003 for source diversity, IMG-014 for placeholder detection. Do NOT satisfy this rule by copying images from another port — that fails IMG-015.
