## Rule
`ai-summary` content length should fall in the 150-250 character range. Under 150 is a warning (not an error) because truncated summaries lose the dual-cap usefulness — the first ~155 chars should be a standalone sentence (see ICP-002), and going too short removes context.

## Why (rationale)
150-250 is the sweet spot where the summary can be a full complete first sentence plus a brief additional detail. Below 150, either the first sentence is stripped of meaning or there's no second clause at all. Validator flags as WARNING at this range so writers see the signal without being blocked.

## Pass example
```html
<meta name="ai-summary" content="Nassau is a tender-free port five minutes walk from downtown Bay Street. Jitneys to Cable Beach $1.25; Atlantis taxi $8 one-way; Queen's Staircase a short walk."/>
```
Length: ~195 chars. Target range satisfied.

## Fail example
```html
<meta name="ai-summary" content="Nassau cruise port guide."/>
```
Length: 26 chars. Validator emits: `ai-summary is short (26 chars), recommended 150-250`.

## Fix guidance
Expand the summary to describe the page with specific facts — distances, prices, names, conditions. A 150+ char summary naturally includes a standalone first sentence and then useful specifics.
