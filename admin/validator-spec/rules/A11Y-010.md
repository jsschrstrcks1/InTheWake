## Rule
The visible breadcrumb trail must be contained in a `<nav>` element with `aria-label="Breadcrumb"` (exact string). Screen readers use the label to distinguish this nav region from the site's primary nav.

## Why (rationale)
Pages typically have multiple `<nav>` landmarks — primary, breadcrumb, footer, maybe a secondary section nav. Without distinct aria-labels, screen reader users hear "navigation" four times and have no way to tell them apart. `"Breadcrumb"` (capitalized, exact) is the conventional value.

## Pass example
```html
<nav aria-label="Breadcrumb">
  <ol class="breadcrumbs">
    <li><a href="/">Home</a></li>
    <li><a href="/ports.html">Ports</a></li>
    <li aria-current="page">Nassau</li>
  </ol>
</nav>
```

## Fail example
```html
<nav class="breadcrumbs">
  <ol>...</ol>
</nav>
<!-- or -->
<div class="breadcrumbs">
  <ol>...</ol>
</div>
```
Validator emits: `Gold standard gap: missing breadcrumb navigation (aria-label="Breadcrumb")`.

## Fix guidance
Use `<nav aria-label="Breadcrumb">`. The JSON-LD BreadcrumbList (SCHEMA-005) is a separate concern — both are required, neither substitutes for the other.
