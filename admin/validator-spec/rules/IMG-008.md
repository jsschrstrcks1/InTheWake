## Rule
The hero section (`section.port-hero`, `#hero`, or `.port-hero`) must be inside a `<main>`, `<article>`, or `.card` container — NOT at the top level of `<body>` before the header/nav.

## Why (rationale)
A hero placed outside main breaks semantic landmarks: screen readers land on the nav instead of the main content, and the "skip to main content" link (A11Y-001) skips over the hero entirely. The header/nav/hero/main order must be: header → nav → main (which contains hero + body + footer).

## Pass example
```html
<body>
  <a href="#main" class="skip-link">Skip to main content</a>
  <header>...</header>
  <nav>...</nav>
  <main id="main">
    <article>
      <section class="port-hero">
        <img src="..." alt="..."/>
      </section>
      <section>...</section>
    </article>
  </main>
</body>
```

## Fail example
```html
<body>
  <section class="port-hero"><img .../></section>  <!-- BAD: top-level, before header -->
  <header>...</header>
  <main>...</main>
</body>
```
Validator emits: `Hero section must be inside main content area (article/card), not at top of body before header`.

## Fix guidance
Move the hero section inside `<main>`. If the visual effect needs full-bleed width, use CSS (e.g., `.port-hero { margin-left: calc(50% - 50vw); width: 100vw; }`) — don't pull the element out of the semantic hierarchy.
