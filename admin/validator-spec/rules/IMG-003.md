## Rule
Alt text shorter than 20 characters is warned (not blocked). A 3-word generic alt like "port image" technically passes IMG-001 but carries almost no information to screen readers. 20 chars is a lenient floor — real descriptive alts are usually 50-150.

## Why (rationale)
IMG-001 only checks presence. A missing alt is a failure; a too-short alt passes that check but fails the WCAG spirit. IMG-003 is the next bar up: "it has some content, but it might be junk." Warning level keeps it from blocking commits while creating visibility.

## Pass example
```html
<img src="..." alt="Royal Caribbean ship docked at Prince George Wharf with downtown Nassau visible in background"/>
```
Length: 95. Passes.

## Fail (warning) example
```html
<img src="..." alt="port"/>
<img src="..." alt="image"/>
<img src="..." alt="nassau"/>
```
Validator emits: `N images have alt text shorter than 20 characters`.

## Fix guidance
Describe what the image shows. "Nassau" is a caption; "Colonial architecture on Bay Street, central Nassau, at midday" is an alt. If the image is purely decorative, use `alt="" aria-hidden="true"` — that also satisfies IMG-001 and doesn't trigger this warning.
