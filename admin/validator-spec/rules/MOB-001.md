## Rule
Interactive elements (buttons, links, form controls, anything with `role="button"`) must have a minimum tap area of 44×44 CSS pixels. Inline-styled explicit widths or heights below 44px on interactive elements are flagged.

## Why (rationale)
WCAG 2.5.5 (AAA-level, but adopted site-wide for real-world accessibility). Small targets fail on mobile — they fail for users with motor impairments, for older readers, for anyone using a phone on a moving ship. The mobile validator (`admin/validate-mobile-readiness.js`) checks inline-style overrides that force small targets.

## Pass example
```html
<button class="cta" style="padding: 12px 20px; min-height: 44px;">Read our Nassau guide</button>
<a class="icon-button" aria-label="Share this page" style="width: 44px; height: 44px;">...</a>
```

## Fail example
```html
<a style="height: 28px; display: inline-block; padding: 2px;">Click here</a>
<button aria-label="Close" style="width: 24px; height: 24px;">×</button>
```
Validator emits: `<button> "Close" has inline height: 24px (min 44px required)` (line 326-344 of validate-mobile-readiness.js).

## Fix guidance
Remove the inline-style height/width constraint and let the shared button CSS take over. If a visually-smaller icon is required, keep the tap area 44×44 and make the visible icon smaller with padding (e.g., `height: 44px; padding: 10px;` produces a 24×24 icon inside a 44×44 target).
