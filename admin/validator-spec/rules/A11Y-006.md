## Rule
Interactive controls without visible text content (icon-only buttons, print button, share button, close button) must declare an `aria-label` or `aria-labelledby` attribute so screen readers announce the button's purpose.

Validator today specifically checks print buttons; the principle applies to any iconographic control.

## Why (rationale)
Screen readers announce a button's accessible name. With visible text, the text is the name. With only an icon, there's nothing to announce unless `aria-label` provides one. A naked `<button>🖨</button>` reads as "button" — useless for blind users.

## Pass example
```html
<button aria-label="Print this page">🖨</button>
<button aria-label="Close dialog"><svg>...</svg></button>
<button>Read our Nassau guide</button> <!-- text-based; no aria needed -->
```

## Fail (warning) example
```html
<button onclick="window.print()">🖨</button>
```
Validator emits: `Print button missing aria-label attribute`.

## Fix guidance
Add `aria-label="..."` with a short phrase describing the action. Don't duplicate visible text (that creates noise). Keep labels action-oriented: "Print this page", not "Print button".
