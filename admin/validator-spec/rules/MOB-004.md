## Rule
Tables with multiple columns of data (e.g., deck plans, fleet specs, pricing matrices) must have a mobile-overflow strategy: either wrap in `.table-responsive` / `overflow-x: auto` container, OR apply a mobile breakpoint that transforms the table into a stacked list. Unwrapped wide tables fail at warn.

## Why (rationale)
A 6-column spec table doesn't fit on a 375px phone viewport. Unwrapped, it either overflows horizontally (breaking page scroll) or compresses each column to unreadable narrowness. The fix is standard (`overflow-x: auto` wrapper) but easy to forget.

## Pass example (horizontal scroll container)
```html
<div class="table-responsive" style="overflow-x: auto;">
  <table>
    <thead><tr><th>Deck</th><th>Class</th><th>Category</th><th>Sqft</th><th>Balcony</th><th>Price</th></tr></thead>
    <tbody>...</tbody>
  </table>
</div>
```

## Pass example (stacking via CSS)
```html
<table class="mobile-stack">...</table>
<style>
  @media (max-width: 600px) {
    .mobile-stack thead { display: none; }
    .mobile-stack tr { display: block; margin-bottom: 1rem; }
    .mobile-stack td { display: block; }
    .mobile-stack td::before { content: attr(data-label) ": "; font-weight: bold; }
  }
</style>
```

## Fail (warning) example
```html
<table>
  <thead><tr><th>Deck</th><th>Class</th>...</tr></thead>
  <tbody>...</tbody>
</table>
<!-- no wrapper, no breakpoint — overflows on mobile -->
```

## Fix guidance
Wrap the table in `<div class="table-responsive" style="overflow-x: auto;">`. The wrapper is a single-line fix that works on every table. If the table genuinely warrants a stacked mobile layout, use the data-label pattern shown above.
