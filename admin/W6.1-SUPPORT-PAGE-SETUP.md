## What you need to do to activate

### Step 1 — Set up the Buy Me A Coffee account

1. Go to https://buymeacoffee.com and create an account
2. Choose a handle. **The current placeholder is `inthewake`** (from URL `https://buymeacoffee.com/inthewake`). If a different handle is taken or preferred, update accordingly.
3. Configure two payment surfaces:
   - **Tip jar** (for W6.1a): default amounts $3, $5, $10, $25
   - **Membership** (for W6.1b — the Duck Club): tier at $3/month or $5/month, or annual at $35/year or $60/year
4. Brand the BMAC page to match (compass-rose icon, calm color palette, link back to cruisinginthewake.com)

### Step 2 — Swap the placeholder URLs

Two URLs in `/support.html` currently point to placeholders. Once your BMAC handle is set up, update:

```
File: support.html
Line ~227 (tip jar button):
  href="https://buymeacoffee.com/inthewake"
  → replace with your actual BMAC URL

Line ~244 (Duck Club button):
  href="https://buymeacoffee.com/inthewake/membership"
  → replace with your actual Duck Club membership URL
```

If the chosen handle differs from `inthewake`, both URLs need updating.

### Step 3 — Test the flow

1. Visit `https://cruisinginthewake.com/support.html` in a fresh browser
2. Click "Tip via Buy Me A Coffee" — should redirect to your tip page
3. Click "Join the Duck Club" — should redirect to your membership page
4. Make a $1 test transaction with your own card to confirm the full flow works (refundable through BMAC if you want)

### Step 4 — Set up the monthly footer-thanks process

This is the recurring lightweight workstream needed to fulfill the Duck Club promise of monthly recognition:

1. Once a month, log into BMAC, get the list of supporters (names or handles, not full identities) for the past 30 days
2. Update a small file like `admin/data/supporters-current.json` with the names
3. The footer can render this as a simple "Thanks to this month's supporters: …" block on the homepage, support.html, or wherever you choose
4. **Initially low-volume**: probably 1–10 people, manageable manually. If volume grows, a small automated sync is reasonable.

### Step 5 — Set up the sticker logistics

For the optional annual Duck Club sticker:

1. Order a small batch of duck-themed stickers (Stickermule, Sticker Mule, or similar — ~$50 for 50)
2. Once a year, send to active Duck Club members who opted to receive one
3. Collect mailing addresses through the BMAC membership signup form (BMAC supports custom signup questions)

## Decisions still open (not blocking, can be addressed later)

- **Which BMAC handle?** Default placeholder is `inthewake` — you may want something different
- **What pricing tier(s)?** The plan says $3–5/month or $35–60/year — pick what feels right
- **Should the Duck Club have ANY sub-tiers?** (e.g., "First Mate" at $5, "Captain" at $10). The plan recommends keeping it simple and flat to honor the no-feature-gate principle.
- **Do you want to add the support link to the homepage hero?** Currently it's footer-only. A subtle link near the trust badge or a `/support` mention in the welcome message would increase visibility, but the plan was conservative on this.

## Reminders about what NOT to do

- Don't put the Support link on `/solo/` or grief-adjacent pages — the script enforces this, but if you add manual links elsewhere, preserve the bright line
- Don't gate any existing tool, port page, ship guide, or article behind a Duck Club tier — the brand promise is permanent
- Don't add "early access" to existing content — the Duck Club promise is "early heads-up on new tools," which means notification, not access lockout
- Don't add urgent-language CTAs anywhere on `/support.html` ("Sign up now!", "Limited spots!"); the page is calm by design

## Migration script usage (for future updates)

The script at `admin/scripts/footer-rollout.py` is the source of truth for sitewide footer behavior. It's idempotent and safe to re-run:

```bash
# Dry run — show what would change, no writes
python3 admin/scripts/footer-rollout.py

# Apply changes
python3 admin/scripts/footer-rollout.py --apply

# Limit scope to a specific glob
python3 admin/scripts/footer-rollout.py --apply --glob "ports/*.html"
```

If you create new HTML pages later, run the script in dry-run mode to verify they pick up the standard footer; if not, the script will report them as "unrecognized variants" and you can either add the pattern or update the new file's footer to match.
