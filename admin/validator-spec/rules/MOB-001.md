## Rule
The viewport meta must (1) exist, (2) include `width=device-width`, and (3) include `initial-scale=1`. Missing tag, missing either option, or presence of `user-scalable=no` (which disables pinch-zoom) each produce a distinct failure.

Tracked as validator's MOB-001 at lines 85-135. Overlaps with A11Y-003 which covers the same tag from the accessibility angle — both rules cite the same validator check.

## Why (rationale)
Without `width=device-width`, mobile browsers render at a simulated 980px and zoom-fit, producing tiny text. Without `initial-scale=1`, initial zoom is browser-dependent. With `user-scalable=no`, low-vision users who need to pinch-zoom are blocked — an accessibility failure.

## Pass example
```html
<meta name="viewport" content="width=device-width, initial-scale=1"/>
```

## Fail examples
```html
<!-- missing entirely -->
<!-- BLOCKING: 'Missing viewport meta tag' -->

<meta name="viewport" content="initial-scale=1"/>
<!-- BLOCKING: 'Viewport meta tag missing "width=device-width"' -->

<meta name="viewport" content="width=device-width"/>
<!-- BLOCKING: 'Viewport meta tag missing "initial-scale=1"' -->

<meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no"/>
<!-- WARNING: 'Viewport meta tag disables user scaling (user-scalable=no). This harms accessibility.' -->
```

## Fix guidance
Use the canonical content string `"width=device-width, initial-scale=1"`. Do not add `user-scalable=no`, `maximum-scale=1`, or anything else that prevents users from zooming.
