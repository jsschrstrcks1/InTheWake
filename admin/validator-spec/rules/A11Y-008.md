## Rule
The previous/next navigation buttons inside a carousel must declare `aria-label` ("Previous slide", "Next slide"). This is a special case of A11Y-006 (icon-only buttons) scoped to the carousel pattern.

## Why (rationale)
Carousel nav buttons are almost always icon-only (chevrons, arrows). Without aria-label they announce as unnamed buttons. Users navigating by Tab have no idea what each button does until they activate it.

## Pass example
```html
<div class="swiper" aria-label="Photo gallery">
  <div class="swiper-slide">...</div>
  <button class="swiper-button-prev" aria-label="Previous photo"></button>
  <button class="swiper-button-next" aria-label="Next photo"></button>
</div>
```

## Fail (warning) example
```html
<button class="swiper-button-prev"></button>
<button class="swiper-button-next"></button>
```
Validator emits: `Carousel navigation button missing aria-label`.

## Fix guidance
Add `aria-label` to both prev and next buttons. Consistent labels across carousels ("Previous slide"/"Next slide") are fine; content-specific labels ("Previous photo"/"Next deck plan") are better.
