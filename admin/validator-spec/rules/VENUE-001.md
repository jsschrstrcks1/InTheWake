## Rule
A venue page's JSON-LD `mainEntity` must declare a Schema.org type that matches the real category of the venue:
- **Dining venues** (restaurants, specialty dining, bars) → `Restaurant`, `Bar`, `NightClub`, or `CafeOrCoffeeShop` as appropriate
- **Activity venues** (climbing walls, escape rooms, mini golf, water park features) → `SportsActivityLocation`, `AmusementPark`, `AmusementParkAttraction`
- **Shopping venues** → `Store` or `ShoppingCenter`

Using `Restaurant` for a climbing wall is a failure. The March 2026 audit found 14 activity venues mistyped as Restaurants.

## Why (rationale)
Schema.org typing drives rich results, AI summaries, and voice-assistant responses. A climbing wall returned as "a restaurant" to a voice query is misinformation generated by the site's own data. Users who ask an AI assistant "what restaurants are on Icon of the Seas" shouldn't get back "FlowRider, Ultimate Abyss, and Rock Climbing Wall."

## Pass example
```html
<!-- Chops Grille (steakhouse) -->
<script type="application/ld+json">
{ "@context": "https://schema.org", "@type": "WebPage",
  "mainEntity": {
    "@type": "Restaurant",
    "servesCuisine": "Steakhouse",
    "name": "Chops Grille"
  }
}
</script>

<!-- Ultimate Abyss (10-story dry slide) -->
<script type="application/ld+json">
{ "@context": "https://schema.org", "@type": "WebPage",
  "mainEntity": {
    "@type": "AmusementParkAttraction",
    "name": "Ultimate Abyss"
  }
}
</script>
```

## Fail example
```html
<!-- Ultimate Abyss — wrongly typed as Restaurant -->
<script type="application/ld+json">
{ "@context": "https://schema.org", "@type": "WebPage",
  "mainEntity": { "@type": "Restaurant", "name": "Ultimate Abyss" }
}
</script>
```
This is the exact class of error VENUE_PAGE_AUDIT_2026_03_04 found on 14 pages.

## Fix guidance
Open the venue's real nature: does someone eat there, play there, or shop there? Map to the appropriate Schema type. When in doubt, a specific parent type (`SportsActivityLocation`) is better than a wrong sibling (`Restaurant`). Never leave `@type: WebPage` as the only declaration on a venue page — that fails SCHEMA-001 too.
