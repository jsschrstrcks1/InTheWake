## Why this list matters

The user's concern when authorizing this spec: *"500 individual rules that don't ever get called."*

This file is the answer. Every rule that isn't wired to a live validator check is visible here. If the file grows past ~30 entries, something is wrong — either the spec is drifting toward wishful thinking, or validator work has fallen behind. Either way, visible.

Nightly (or pre-commit) running of `find-orphans.cjs` enforces this.

---

## Exit conditions for a rule on this list

A rule leaves `ORPHANS.md` when:

1. Validator code is written that enforces it → rule's `implementation:` field updated with file+function+lines → file regenerates without the rule. OR
2. The rule is deprecated (`status: deprecated`) with a CHANGELOG entry explaining why.

A rule should not simply linger on ORPHANS indefinitely.

---

## find-orphans.cjs exit codes

- `0` — clean. Either no orphans, or every orphan is explicitly `provenance: S-only` (acceptable).
- `1` — data-integrity failure. A `V-only` rule has `implementation: none` (contradiction), or a rule file is missing required fields, or `check: TBD` appears anywhere.

CI should fail on exit code 1.
