## Rule
JavaScript source must not contain `debugger` statements. Pattern match on `\bdebugger\b`.

## Why (rationale)
A `debugger` statement pauses execution when DevTools is open. Shipping one to production means any visitor who opens DevTools (including curious cruisers debugging why a tool isn't loading) gets unexpectedly interrupted. Also: its presence is a clear tell that code was pushed mid-debug-session without cleanup.

## Pass example
```js
function calculateTotal(packages) {
  return packages.reduce((sum, p) => sum + p.price, 0);
}
```

## Fail example
```js
function calculateTotal(packages) {
  debugger; // BLOCKING
  return packages.reduce((sum, p) => sum + p.price, 0);
}
```
Validator emits: `Remove debugger statements`.

## Fix guidance
Delete the line. If you need a conditional pause for development, use a constant feature flag and guard it explicitly (`if (window.__DEBUG) { debugger; }`) — but even then, the flag must be absent from production bundles. Simplest: delete.
