## Rule
When a port page contains more than 25 images, the validator emits a warning. Not a failure — pages can exceed 25 if the content genuinely needs them — but the warning exists because past audits found pages padded with marginal images to hit a (non-existent) "more is better" bar.

## Why (rationale)
Galleries with >25 images usually include several that don't contribute unique information. Twenty-five is a soft ceiling that encourages curation. Pages can legitimately exceed it (a port with six distinct neighborhoods, each with 5 images = 30) and the warning doesn't block them.

## Pass example
Port page with 22 images. Passes.

## Fail (warning) example
Port page with 32 images, many of which are similar shots of the same harbor. Validator emits: `Page has 32 images, recommended maximum is 25`.

## Fix guidance
If you see this warning, review the gallery for near-duplicates. Keep the one that conveys the most information; drop the rest. If all 32 are genuinely distinct and worthwhile, the warning can be left in place — it's advisory.
