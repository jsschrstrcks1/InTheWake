## Rule
A page must contain exactly one `<meta name="ai-summary">` tag. Zero is caught by ICP-001; more than one fails this rule. Currently enforced only in `admin/validate-ship-page.js` (line 540-542); port and venue validators should adopt this check.

## Why (rationale)
Two ai-summary tags is ambiguity: which one do AI consumers trust? Duplicate tags are almost always an editing accident (a previous version wasn't removed when the new one was added). The check is trivial to add; the silent failure mode of duplicates is not.

## Pass example
```html
<meta name="ai-summary" content="Nassau is a tender-free port..."/>
```

## Fail example
```html
<meta name="ai-summary" content="Old summary from February."/>
<meta name="ai-summary" content="Updated summary from April."/>
```
Validator emits: `Duplicate ai-summary meta tags`.

## Fix guidance
Delete all but one. Keep the most recent, most accurate version.

## Backfill note
Port and venue validators should add the equivalent check. Spec flags `standards-backfill: yes` so Phase 6 regenerated standards docs include "exactly one ai-summary" alongside the existing "ai-summary required" rule.
