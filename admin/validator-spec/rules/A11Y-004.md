## Rule
Every page must have an element with `id="main-content"`. This is the target of the skip-link (A11Y-001). Typically this is the `<main>` element itself, but any landmark-role element with this id satisfies the check.

## Why (rationale)
Without `#main-content`, the skip-link has nowhere to go — users pressing Tab land on it, activate it, and nothing happens. The two rules are paired: A11Y-001 requires the skip-link, A11Y-004 requires the landmark it targets.

## Pass example
```html
<body>
  <a href="#main-content" class="skip-link">Skip to main content</a>
  <header>...</header>
  <nav>...</nav>
  <main id="main-content">
    ...
  </main>
</body>
```

## Fail example
```html
<body>
  <a href="#main-content" class="skip-link">Skip to main content</a>
  <main>
    <!-- no id on main; skip-link broken -->
  </main>
</body>
```
Validator emits: `Missing id="main-content" section`.

## Fix guidance
Add `id="main-content"` to the `<main>` element. If for some reason you need a different target (content inside main, below a hero), move the id to that inner element and update the skip-link href to match.
