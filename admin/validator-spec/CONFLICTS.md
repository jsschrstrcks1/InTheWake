## How conflicts are surfaced

During Phase 2 (validator → spec) and Phase 3 (standards → spec), whenever a rule already in the spec acquires a second source that disagrees with the first, the rule file is updated:

- `provenance:` changes to `V-S-conflict`
- `decision:` set to `UNRESOLVED`
- `## Implications` section added with both-directions analysis
- `## Recommendation` section added with author's suggestion + justification

Regenerate this file with `node scripts/generate-conflicts.cjs`.

## User review protocol (Phase 5)

1. Read this file top to bottom.
2. For each conflict, read the rule file linked.
3. For each rule, either:
   - **Accept** the recommendation → edit the rule file, change `decision: UNRESOLVED` → `decision: FINAL`, save.
   - **Override** → edit the rule file, update the rule body to reflect the decision, change `decision: FINAL`. Leave the author's recommendation in place as history.
4. After all conflicts have `decision: FINAL`, regenerate this file. It should be empty (except for the front-matter).
5. Only then does Phase 6 (regenerate standards) begin.
