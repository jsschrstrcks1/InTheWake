## Rule
Every port page must contain a hero section, identified by one of: `section.port-hero`, `#hero` element, or any element with class `port-hero`. Absence of all three fails as BLOCKING.

## Why (rationale)
The hero is the single most-viewed element on any port page — it's the first fold, the first impression, the hook that convinces a reader to stay. Every port page standard demands one. A page that doesn't have a hero isn't really a port page; it's a structured-content stub.

## Pass example
```html
<main>
  <article>
    <section class="port-hero">
      <img src="..." alt="..."/>
      <div class="port-hero-overlay"><h1>Nassau, Bahamas</h1></div>
    </section>
    ...
```

## Fail example
```html
<main>
  <article>
    <h1>Nassau, Bahamas</h1>
    <p>Welcome to Nassau...</p>
    <!-- no .port-hero, #hero, or section.port-hero anywhere -->
```
Validator emits: `Page must have a hero box section with class "port-hero"`.

## Fix guidance
Add the hero section. Use `<section class="port-hero">` as the canonical form. This rule is the gatekeeper for IMG-008 through IMG-010 — without the hero section those rules never fire, but this rule catches the absence first.
