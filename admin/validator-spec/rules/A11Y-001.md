## Severity drift discovered in Phase 2 batch 4

Port validator (line 3729-3730) emits WARNING on missing skip-link. Ship validator (line 994-996) emits BLOCKING on missing skip-link. Both validators are checking the same rule; the severity disagreement is arbitrary — exactly the V-V drift the spec catches.

See IMG-004 for the identical pattern on lazy-loading. User accepted "promote to BLOCKING" there; same reasoning applies here (WCAG 2.1 AA requirement, real a11y impact, bounded fix cost).

## Implications

### If port's WARNING wins
WCAG Success Criterion 2.4.1 is a required A-level criterion; relaxing the rule to match port's warning softens a hard a11y requirement. Not recommended — this is the current anomalous state, not a defensible position.

### If ship's BLOCKING wins
- Consistent severity across validators, proper WCAG enforcement.
- Port validator line 3729-3730 promoted from `warnings.push(...)` to `errors.push(...)`.
- Audit needed: how many port pages currently pass via warning but would fail under blocking? Fallout: one-line fix per offending page (add `<a href="#main-content" class="skip-link">`).
- Same pattern as IMG-004 (which user already accepted "promote to BLOCKING" on).

## Recommendation
**Promote port to BLOCKING to match ship.** WCAG 2.4.1 is a level-A criterion — strictest real accessibility bar. The current port severity is the anomaly, not the standard. Same pattern user accepted on IMG-004.

## Rule
Every page must have a "Skip to main content" link as the first focusable element, visually hidden until focused, that jumps to the page's main landmark (`#main` or `<main id="main-content">`).

## Why (rationale)
WCAG 2.1 AA SC 2.4.1 (Bypass Blocks). Keyboard and screen-reader users should not have to tab through 30 nav links on every page. The site serves disabled travelers — this is non-optional.

## Pass example
```html
<body>
  <a href="#main" class="skip-link">Skip to main content</a>
  <header>...</header>
  <nav>...</nav>
  <main id="main">...</main>
</body>
```

## Fail example
```html
<body>
  <header>...</header>  <!-- skip-link absent -->
  <nav>...</nav>
  <main>...</main>
</body>
```
Validator emits `missing_skip_link: Missing skip link (recommended for accessibility)` at warn severity.

## Fix guidance
Add a visually-hidden anchor as the first body child. CSS: `.skip-link { position: absolute; top: -40px; left: 0; } .skip-link:focus { top: 0; }`. Target must match the main element's id.
