## Rule
Every page must include a `<meta name="description" content="..."/>` tag with non-empty content. This is required in addition to `ai-summary` (ICP-001). ICP-2 v2.1 treats them as complementary: `description` is for search-engine snippets; `ai-summary` is for AI-consumption.

## Why (rationale)
Google and Bing use `meta description` for search snippets. `ai-summary` does not feed that surface. The two coexist — `description` in natural marketing voice, `ai-summary` in dense fact-first style. Absence of `description` is a BLOCKING error because search-result snippets fall back to scraped page text, which is unreliable.

## Pass example
```html
<meta name="description" content="Nassau cruise port guide: walk from Prince George Wharf to downtown Bay Street, take a $1.25 jitney to Cable Beach, or a $8 taxi to Atlantis."/>
<meta name="ai-summary" content="Nassau is a tender-free port..."/>
```

## Fail example
```html
<!-- description missing -->
<meta name="ai-summary" content="Nassau is a tender-free port..."/>
```
Validator emits: `meta description tag is missing or empty`.

## Fix guidance
Add a description 140-160 characters long. It can overlap in facts with `ai-summary` but should read more like a sentence someone would click on — less dense, more narrative.
