## Rule
Every page must have exactly one `<h1>` element. Zero h1s fails (no top-level heading). Two or more h1s fail (ambiguous page subject).

## Why (rationale)
Screen readers use h1 as the page's canonical "what is this page about" landmark. Multiple h1s confuse the reader-orientation pattern. The most common cause of multiple h1s: a hero overlay text marked as h1 AND a content h1 below it. Validator hint message calls this out: "Fix: change decorative hero text to <p>."

## Pass example
```html
<section class="port-hero">
  <div class="port-hero-overlay">
    <h1>Nassau, Bahamas</h1>  <!-- the one h1 — in hero overlay -->
  </div>
</section>
<main>
  <h2>Quick answers</h2>
  <h2>What the day feels like</h2>
  ...
</main>
```

## Fail example (two h1s)
```html
<section class="port-hero">
  <h1>NASSAU</h1>           <!-- hero h1 -->
</section>
<main>
  <h1>Nassau, Bahamas — Cruise Port Guide</h1>   <!-- content h1 — now there are two -->
</main>
```
Validator emits: `Page has 2 <h1> elements — should have exactly 1. Fix: change decorative hero text to <p>.`

## Fail example (zero h1s)
```html
<main>
  <h2>Nassau, Bahamas</h2>  <!-- top-level heading but as h2 — skips h1 -->
</main>
```
Validator emits: `Page has no <h1> element — every page should have exactly one.`

## Fix guidance
Use exactly one h1. Pick whether it's in the hero overlay or the content body — don't put one in both places. Decorative hero text that isn't the page subject belongs in `<p>` or a `<span>`, not another h1.
